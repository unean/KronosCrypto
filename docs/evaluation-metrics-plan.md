# Kronos 评估指标 — 实施进度与规划

## 背景

Kronos 论文(AAAI 2026, arXiv:2508.02739)的好成绩主要体现在三个维度,而本项目最初的统计页面测的是论文**没有**重点报告的单标的方向命中率 / 点位误差。在 ETH/USDT 15m 上,这些指标接近抛硬币(方向 52%、收盘相关性 0.017),属于 Kronos 的分布外弱项。

本文件记录:把评估指标对齐到论文真正的强项维度。

## 论文强项与可落地指标

| 论文维度 | 性质 | 单标的可做? | 状态 |
|---------|------|------------|------|
| 波动率预测 MAE / R² | 绝对预测 | ✅ | **已完成** |
| RankIC(横截面排序) | 横截面 | ❌ 需多标的 | 规划中 |
| 回测收益(top-k) | 横截面 | ❌ 需多标的 | 规划中 |

## 已完成:波动率预测准确度

**后端** `backend/app/services/snapshots.py` `evaluate()`:
- 新增 `pred_volatility_pct` / `actual_volatility_pct` / `volatility_error_pct`
- 口径:预测序列与实际序列在预测窗口内步进收益的标准差(同一算法,公平对比)

**前端**:
- `frontend/src/utils/stats.ts` — `aggregate()` 计算波动率 MAE 与 R²(以实际波动为真值)
- `frontend/src/components/StatsPanel.tsx` — 新增「波动率预测 R²」卡片 + 明细表列

**重评说明**:旧快照需重新评估才会带新字段。重评可复用库中已存的 `actual` K 线,无需联网。

### 首次结果(54 个 ETH/USDT 15m 快照)
- 平均预测波动 0.090% vs 实际波动 0.222% → 模型**系统性低估**波动
- 波动率 MAE 0.140%,R² **-1.23**(<0,比直接猜历史均值还差)

结论:在 ETH 15m 上,即便是论文强项维度也未复现。进一步印证 crypto 短周期是分布外场景。下一步价值在于**换长周期(4h/1d)** 与**多标的横截面**重新检验。

## 下一步规划

### 1. RankIC(横截面排序相关性)— 优先级最高

论文 93% 提升的真正来源。Kronos 的设计意图是"选币排序器",而非单币预言机。

**改造点**:
- 后端:支持同一锚点时间对**多个 symbol** 批量打快照(新增批量预测端点或锚点字段)
- 评估:按锚点聚合同一时刻的多标的快照,用 `pred_return_pct` 排序 vs `actual_return_pct` 排序,算 Spearman 相关(IC)及其均值/IR
- 前端:统计页新增 RankIC 卡片与按锚点的 IC 时间序列

### 2. 回测收益(top-k 策略)

RankIC 的延伸,依赖多标的快照。
- 每个锚点选预测收益最高的 k 个标的,等权买入,持有至预测窗口结束
- 计算累计收益、年化超额收益(AER)、信息比率(IR),对标基准(等权持有全部 / BTC)
- 前端:累计收益曲线 vs 基准
